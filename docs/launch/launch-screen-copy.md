## 1. App 啟動畫面（Launch Screen）

### 1.1 主版本（適用 Lottie 或 static 都可）

```

朵朵陪妳記錄妳的身體節律
```

**設計指引**：
- 背景：peach 漸層（peach-100 → peach-200）
- 朵朵居中，size 200，outfit=`fp_crown`，mood=`gentle`
- 主標 `font-display` 32pt，peach-500

- Lottie 版本：朵朵眨眼 + floaty 上下浮動 1 個循環即進入主畫面

### 1.2 簡版（fallback / static PNG）

```

🌙
```

- 用於低階設備或 splash duration 太短時
- 朵朵 + 月亮 emoji + App 名單列即可

## 2. Onboarding 第一句歡迎詞（A/B Test 用 3 版本）

### Version A — 隱私優先（保守安全）

```
嗨，我是朵朵 🌙

我會陪妳一起記錄妳的身體節律。
妳的資料只屬於妳，
我不會賣給任何人。

慢慢來，我們一起。
```

**測試假設**：擔心隱私的用戶（看過 Flo 新聞）會優先轉換。

### Version B — 陪伴優先（情感連結）

```
嗨，妳來了 🌸

我是朵朵，
從今天開始，我會在這裡。

不催妳每天打卡，
忘了一天，我幫妳記著。

我們慢慢來。
```

**測試假設**：被「daily streak guilt」打到怕的用戶會被打中。

### Version C — 功能優先（直接賣點）

```
嗨，朵朵在這 🦋

接下來幾分鐘，
我會幫妳設定：
· 妳的週期長度
· 上次經期的第一天
· 想記錄的內容

設定完，妳的月曆就準備好了。
```

**測試假設**：來自搜尋「經期 App」「排卵記錄」的高意圖用戶會喜歡這版（直接進主題）。

### 三版共用 CTA Button

```
一起開始 →
```

> ❌ 不寫「立即開始」「註冊」「Sign Up」
> ✅ 「一起開始」對齊集團 tone（陪伴感）

## 3. Push Notification 範本（5 個情境）

> ⚠️ 集團硬規則：推播一律是「朵朵想跟妳說」型，不寫「您」、不寫「立即」、不放醫療詞
> ⚠️ Onboarding 完即詢問推播權限；用戶可隨時關閉
> ⚠️ 推播頻率上限：每日不超過 1 則（避免疲勞）

### #1 經期預測提醒（pre-period）

**Title**: `朵朵想跟妳說 🌙`
**Body**: `經期大概還有 3 天 🌸 朵朵幫妳備好月曆了，要不要看一眼？`

**觸發**：預測經期前 3 天 09:00（用戶時區）

### #2 經期當天提醒

**Title**: `朵朵在這 🌙`
**Body**: `朵朵猜今天可能來了。如果到了，記得點一下 — 我幫妳記著。`

**觸發**：預測經期當天 10:00

### #3 連續記錄鼓勵（streak gentle，不 guilt-trip）

**Title**: `朵朵替妳開心 🌸`
**Body**: `妳已經跟朵朵走了 7 天了。慢慢來，朵朵都看在眼裡。`

**觸發**：連續記錄 7 天當晚 21:00

> ❌ 不寫「streak 中斷」威脅型
> ✅ 「慢慢來」「都看在眼裡」陪伴型

### #4 排卵窗口提醒（BBT / 備孕族群）

**Title**: `朵朵悄悄話 🦋`
**Body**: `朵朵看到妳的 BBT 開始變化，這幾天可能是排卵窗口。`

**觸發**：BBT 偵測到排卵 shift 當天 12:00

### #5 找回提醒（re-engagement，極度溫和）

**Title**: `朵朵在等妳⋯ 🌙`
**Body**: `好幾天沒見啦。妳還好嗎？有空回來看看，朵朵在這裡。`

**觸發**：用戶 5 天未開 App 後 19:00 推播一次（之後 14 天不再推，避免騷擾）

## 4. 推播文案紅線

❌ 不寫：

- 「您」「會員」「您的訂閱」
- 「立即」「馬上」「快來」
- 「您的經期延遲了 X 天」（焦慮型）
- 「streak 中斷了」（guilt-trip）
- 「特價」「限時」「優惠」（行銷詞）
- 任何商品名 / 加盟暗示
- 任何療效詞（治療 / 改善 / 緩解 / 修復）

✅ 寫：

- 「朵朵想跟妳說」「朵朵悄悄話」「朵朵在這」
- 「妳」「朋友」
- 「慢慢來」「沒關係」「妳還好嗎」
- 「要不要⋯？」「想⋯嗎？」（邀請型，不是命令型）
- 一個 emoji 點綴，不過量

## 5. 系統訊息 / Toast / Empty State 文案

### 5.1 Toast（操作回饋）

| 情境 | 文案 |
|---|---|
| 記錄成功 | 「記下來了 🌸」 |
| 記錄失敗 | 「朵朵這邊有點卡，再試一次？」 |
| 網路斷線 | 「朵朵收不到網路訊號了，等等再來」 |
| 已同步 | 「朵朵幫妳同步好了 ✨」 |
| 同步中 | 「朵朵正在幫妳整理⋯」 |

### 5.2 Empty State（沒資料時）

| 畫面 | 文案 |
|---|---|
| 月曆首次打開 | 「還沒有記錄。從今天開始，朵朵陪妳一起 🌙」 |
| 症狀記錄空 | 「今天還沒記錄症狀。如果有不舒服或想記下的，點一下吧。」 |
| 衛教文章未解鎖 | 「Premium 的朵朵會帶妳看更深的內容，要不要試試看？」 |
| BBT 數據不足（< 7 天） | 「朵朵需要再陪妳記 {n} 天，才能看出妳的雙相曲線。慢慢來。」 |

### 5.3 Error Message（錯誤訊息）

| 情境 | 技術錯誤 ❌ | 朵朵語氣 ✅ |
|---|---|---|
| 登入失敗 | Authentication failed | 朵朵找不到妳，再試一次？ |
| 密碼錯誤 | Incorrect password | 密碼好像不太對，朵朵記得是另一組？ |
| Server error 500 | Internal Server Error | 朵朵這邊出了點狀況，等一下再來看看？ |
| Timeout | Request timeout | 朵朵等太久了，網路是不是慢慢的？ |
| 訂閱失敗 | Payment failed | 付款沒過，朵朵幫妳保留訂閱頁，妳隨時可以再試。 |

## 6. 引導文案（onboarding 細項）

### 6.1 設定週期長度

```
妳的週期通常幾天？
（不知道也沒關係，朵朵會用 28 天當起點，記錄久了會更準）
```

### 6.2 上次經期第一天

```
上次經期的第一天是什麼時候？
（記不清楚？選一個最接近的，朵朵幫妳調整）
```

### 6.3 推播權限要求

```
要朵朵在重要的日子悄悄提醒妳嗎？
· 經期前 3 天
· 排卵窗口
· 連續記錄的小鼓勵

可以隨時關掉。
```

### 6.4 健康資料同意

```
妳的健康資料只存在妳的帳號裡，
不會賣給任何人，不會用做廣告。

要不要一起開始？
```

CTA: `好的，一起 →` / `先看看再說`
